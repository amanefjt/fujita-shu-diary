## 2026-04-01: 初期プロジェクト解析と展開計画
- **基本機能**: 既存の `my-simple-diary`（Hugo製）をベースにする。
- **ターゲットユーザー**: 友人（猪本典子さん）など、他の人でも簡単に使えるようにしたい。
- **運用フロー**:
    - GitHub の特定フォルダ（`content/diary/`）で新規ファイルを作成。
    - 日付、冒頭の決まった文章（タグや紹介文）、そして Markdown 形式で本文を記述。
    - GitHub Web 編集画面での入力をメインの想定とし、専門的な Git 操作なしで完結させる。
- **技術要件**:
    - Hugo + Markdown。
    - 生成されたサイトが「いい感じ」に見えること。
    - 外部連携（Notionなど）は不要。
- **エージェント管理**: Antigravity のベストプラクティス（Mission, Rules, Skills）を遵守し、`.agent/` 配下に定義する。
- **URLカスタマイズ**: `myouji-namae.pages.dev` など、好きなプロジェクト名で公開するための手順を明記。
- **デザイン**: シンプルでプレミアムな外観（ダークモード、適切な余白等）への拡張。
- **2026-04-01 (続)**: プレミアム化に向けた具体的なデザイン方針の策定（和文タイポグラフィの強化、カード型レイアウトの採用）。
- **2026-04-01 (完)**: プロジェクトを「汎用テンプレート（my-simple-diary-main）」と「個人サイト（fujita-shu-diary）」に分離。
    - **テンプレート**: 誰でもフォークしてすぐに使えるよう、個人データを一切含まないクリーンな状態に整備。
    - **個人サイト**: 名前を伏せつつ、過去の日記（85件）をすべて公開する匿名・個人用サイトとして構築。
    - **BlueSky連携**: 設定ファイルを個人サイトへ移行し、GitHub Secrets による安全な自動投稿環境を構築。
