# 日記の作成・追加手順 (add_diary_entry)

このスキルは、Hugo 形式の日記エントリを新しく作成し、適切なフロントマターを付与するために使用します。

## 実行手順

1.  **ファイル名の決定**: 
    - 今日の日付を `YYYY-MM-DD.md` 形式に変換し、`content/diary/` フォルダ内に作成する準備をします。
2.  **フロントマターの生成**: 
    - 以下の YAML フロントマターを必ず含めます：
    ```markdown
    ---
    title: "YYYY年M月D日（曜） タイトル"
    date: YYYY-MM-DD
    slug: "YYYY-MM-DD"
    ---
    ```
    - 日付とタイトルは、現在の文脈やユーザーの指定に合わせて適切に設定します。
3.  **本文の記述**: 
    - Markdown 形式で内容を正確に書き込みます。
4.  **検証**: 
    - 作成したファイルのパス、およびフロントマターの構文（インデント等）に誤りがないか確認します。

## 注意事項
- **上書きの禁止**: 既に同名の日記ファイルが存在する場合は、勝手に上書きせず、追記するか別名にするか人間に確認してください。
- **匿名性の確保**: ユーザーの要求（タイトルを「日記」にする等）がある場合は、それに従ったタイトルを設定してください。
