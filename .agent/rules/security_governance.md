# セキュリティとガバナンス (Security & Governance)

このプロジェクトでは、利用者のプライバシーとデジタル資産を安全に保護します。

## 1. 機密情報 (Secrets) の保護
- **API キー・トークン**: GitHub Actions 用の BlueSky ハッシュ値や Cloudflare 認証トークンなどを、リポジトリにコミットしない。必ず GitHub の **Secrets**（リポジトリ設定）を通じて管理する。
- **環境変数**: `HUGO_VERSION` など、ビルドに必要な情報のみを Cloudflare Pages の管理画面で定義し、外部への露出を防ぐ。

## 2. プライバシーと匿名性の確保
- **名前の管理**: 個人情報（本名など）は `hugo.toml` の著作者設定や Git のコミット履歴から必要に応じて削除し、匿名化（タイトルを「日記」にする等）を容易にするための仕組みを標準とする。
- **データ分離**: 汎用的なテンプレートと、個人的な日記データを分離して管理し、テンプレート側に個人の足跡を残さない。

## 3. 手順の検証
- **デプロイの安全性**: Cloudflare 側でのビルド・デプロイが成功したことを常に確認し、意図しないビルドエラーによる公開の中断を防ぐ。
- **エビデンスの提示**: AIエージェントが変更を行った後は、ブラウザ検証やコマンドの実行ログなどの証拠（Walkthrough）を提示する。
