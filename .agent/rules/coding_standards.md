# コーディング規約 (Coding Standards)

## 1. テンプレートの保守性
- **Hugo ベストプラクティス**: テンプレート(`layouts/`)、静的資産(`static/`)、設定(`hugo.toml`) の役割分担を明確にする。
- **読みやすさ**: HTML 構造はシンプルに、クラス名や変数は意図が伝わるように記述する。
- **汎用性**: ユーザーが書き換えるべき値（タイトル、著者名、SNSリンク等）を、テンプレート内にハードコードせず、常に `hugo.toml` の `[params]` などから参照する。

## 2. スタイリング (CSS)
- **バニラ CSS**: 原則として外部フレームワーク（Tailwind 等）に依存せず、`static/style.css` で全てのデザインを美しく管理する。
- **モダン・タイポグラフィ**: 和文の美しさを際立たせるフォント選定（Noto Serif JP 等）と、可読性の高いマージン・パディングを維持する。
- **ダークモード**: `@media (prefers-color-scheme: dark)` を活用し、OS の設定に応じたシームレスな切り替えを実現する。

## 3. コンテンツの整合性 (Markdown)
- **フロントマター**: 日記の作成には常に標準的なフロントマター（title, date, slug）を使用し、エージェントが自動生成する。
- **セマンティック HTML**: 適切な見出し構造（h1, h2, h3）を守り、アクセシビリティを保つ。
