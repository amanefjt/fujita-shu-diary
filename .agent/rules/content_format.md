# コンテンツフォーマット規約 (Content Format)

## 日記記事の管理ルール

### 1. ファイル名
- **場所**: `content/diary/`
- **形式**: `YYYY-MM-DD.md` (例: `2026-04-01.md`)
- **理由**: 名前順で並べた際に日付順になるようにするため。

### 2. フロントマター (Front Matter)
記事の冒頭には、以下の形式でメタデータを記述する必要があります。

```yaml
---
title: "2026-04-01 の日記"
date: 2026-04-01T21:00:00+09:00
draft: false
---
```

- `title`: 記事のタイトル。通常は日付を推奨。
- `date`: 公開日時（Hugo が並び替えに使用）。
- `draft`: `false` で本番公開。

### 3. 本文の構成
- **冒頭**: 「今日は何をした」などの要約や、特定のタグ（例：#日記 #技術）を推奨。
- **内容**: Markdown 形式で自由に記述。
- **生HTML**: `hugo.toml` の `unsafe = true` 設定により、既存の HTML タグもそのまま使用可能。
